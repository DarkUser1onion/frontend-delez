# Инструкция по запуску Desktop-клиента Delёz

## Системные требования
*   **Операционная система:** Linux (основная разработка), Windows 10/11, macOS 12+.
*   **Node.js:** версия 20 или выше.
*   **Rust:** стабильная версия (устанавливается через [rustup](https://rustup.rs/)).
*   **Git:** для клонирования репозитория.
*   **Зависимости для Linux (Tauri):**
    *   Arch Linux: `sudo pacman -S --needed webkit2gtk-4.1 base-devel curl wget file openssl appmenu-gtk-module gtk3 libappindicator-gtk3 librsvg libvips`
    *   Ubuntu/Debian: `sudo apt install libwebkit2gtk-4.1-dev build-essential curl wget file libssl-dev libgtk-3-dev libayatana-appindicator3-dev librsvg2-dev libvips-dev`
*   **Для голосового ввода на Linux:** `alsa-utils` (пакет, содержащий `arecord`).

## Шаг 1: Клонирование репозитория
Откройте терминал и выполните:
\`\`\`bash
git clone https://github.com/DarkUser1onion/frontend-delez.git
cd frontend-delez/apps/web
\`\`\`

## Шаг 2: Установка зависимостей Node.js
Находясь в папке `apps/web`, выполните:
\`\`\`bash
npm install
\`\`\`
Дождитесь завершения установки. Убедитесь, что в логах нет критических ошибок.

## Шаг 3: Настройка конфигурации
Создайте файл `.env` на основе примера:
\`\`\`bash
cp .env.example .env
\`\`\`
Откройте `.env` в текстовом редакторе и при необходимости укажите реальные значения для `VITE_API_USERNAME` и `VITE_API_PASSWORD`. Остальные переменные можно оставить без изменений.

Пример содержимого `.env.example`:
\`\`\`
VITE_API_USERNAME=change_me
VITE_API_PASSWORD=change_me
VITE_API_URL=
WHISPER_CLI_PATH=/opt/delez/whisper-cli
WHISPER_MODEL_PATH=~/.cache/delez/whisper/ggml-small.bin
\`\`\`

## Шаг 4: Запуск в режиме разработки (dev-режим)
Для запуска приложения с горячей перезагрузкой и инструментами разработчика выполните:
\`\`\`bash
npm run tauri dev
\`\`\`
**Что произойдёт:**
*   Vite запустит dev-сервер на `http://localhost:3000`.
*   Tauri откроет нативное окно приложения, подключённое к этому серверу.
*   В терминале появятся логи запуска Vite и компиляции Rust.
*   Откроется главное окно Delёz. Вы можете пользоваться чатом, дневником, графом и другими функциями.

## Шаг 5 (опционально): Установка как обычное приложение (только Linux)
Скрипт `install-debug.sh` установит приложение в `/opt/delez` и создаст ярлык в меню приложений.
\`\`\`bash
./install-debug.sh
\`\`\`
После установки приложение можно запускать:
*   Через меню приложений (иконка "Delёz").
*   Из терминала: `/opt/delez/run-delez.sh` или `delez`.
**Примечание:** Этот способ использует dev-сервер Vite и предназначен для разработки и тестирования.

## Шаг 6: Сборка AppImage (портативный универсальный запускаемый файл)
\`\`\`bash
./build-appimage.sh
\`\`\`
Скрипт создаст файл `Delez.AppImage` в текущей папке. Запустите его:
\`\`\`bash
./Delez.AppImage
\`\`\`
При **первом запуске** AppImage скачает whisper-cli и модель `ggml-small.bin` в кэш `~/.cache/delez/whisper/`. При повторных запусках загрузка не требуется.

## Проверка голосового ввода на Linux
1.  Откройте приложение (dev-режим или AppImage).
2.  Перейдите в чат.
3.  Нажмите кнопку микрофона (в поле ввода).
4.  Появится тост «Запись 5 сек... Говорите.».
5.  Через 5 секунд тост «Готово», и распознанный текст появится в поле ввода.

Если микрофон не работает:
*   Убедитесь, что `arecord` видит ваш микрофон: `arecord -l`.
*   Проверьте, что `whisper-cli` лежит в `src-tauri/DelezApp/usr/bin/whisper-cli` (для dev) или скачан в `~/.cache/delez/whisper/whisper-cli` (для AppImage).
*   Проверьте модель: `ls -l ~/.cache/delez/whisper/ggml-small.bin`.

## Примечания
*   **Серверный backend** уже должен быть запущен и доступен. По умолчанию desktop-клиент обращается к API через прокси Vite (см. `vite.config.ts`).
*   **Модель whisper** можно скачать заранее:
\`\`\`bash
mkdir -p ~/.cache/delez/whisper
curl -# -L -o ~/.cache/delez/whisper/ggml-small.bin https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-small.bin
\`\`\`
