# Commit Naming Rules

## Формат

```text
<type>(<scope>)<author>: <short summary>

[optional body]

[optional footer]
```

## Поля

- `type` — тип изменения (`feat`, `fix`, `docs`, `chore`, `refactor`, `style`, `test` и т.д.).
- `scope` — область, где произошло изменение, например: `auth`, `api`, `ui`.
- `author` — подпись или ник человека, обязательно в формате `<nickname>`.
- `summary` — краткое описание изменений.
- `body` — подробное объяснение изменений, если оно нужно.
- `footer` — ссылки на задачи, тикеты или дополнительные пояснения.

## Допустимые типы коммитов

| Тип | Для чего используется |
| --- | --- |
| `feat` | Новый функционал |
| `fix` | Исправление бага |
| `docs` | Изменения в документации |
| `style` | Форматирование без изменения логики |
| `refactor` | Рефакторинг без изменения поведения |
| `perf` | Оптимизация производительности |
| `test` | Добавление или изменение тестов |
| `chore` | Вспомогательные изменения: сборка, конфиги, CI/CD |

## Рекомендации

- Использовать английский язык для `type` и `scope`.
- Для `summary` допустим русский или английский язык.
- Подпись автора всегда указывать в формате `<nickname>`.
- Один коммит должен содержать одну логическую задачу.
- Если изменений много, лучше разбить их на несколько коммитов.

## Примеры

```text
feat(auth)<alex>: add refresh token rotation

fix(api)<mike>: исправить обработку 401 для повторного запроса

docs(ui)<kate>: обновить описание страницы авторизации
```
