# Рекомендации по безопасности и эксплуатации – Delёз

1. **Автоматизация резервного копирования**  
   Добавить задание в cron (Linux) или планировщик Windows для ежедневного запуска `backup.sh`/`backup.bat`.

2. **Мониторинг зависимостей**  
   Включить `npm audit` в CI/CD (GitHub Actions, GitLab CI). При обнаружении high-уязвимостей автоматически создавать issue.

3. **Обновление уязвимых пакетов**  
   Запланировать обновление `@langchain/core`, `langsmith`, `uuid` до стабильных версий без breaking changes.

4. **Усиление контроля доступа**  
   Провести ревью всех API-эндпоинтов на наличие IDOR (проверка владельца ресурса). Добавить тесты.

5. **CORS для веб-версии**  
   Если веб-версия будет использоваться, настроить CORS на бэкенде (разрешить только `https://delez.tech`).

6. **Логирование безопасности**  
   Добавить логирование неудачных попыток входа, смены пароля, доступа к чужим данным.

7. **Документация для пользователя**  
   В README добавить раздел «Как сделать backup вручную» с командами.

8. **Подпись установщиков**  
   Убедиться, что .msi и .dmg подписаны (для Windows и macOS). Для AppImage использовать проверку целостности.
