# INCIDENT_REPORT

## 1. Инцидент
Название: Отсутствие проверки API и curl в процессе релиза; нерабочая сборка из-за несовместимости версий LangChain.

## 2. Где обнаружено
При выполнении этапа 6 (CI/CD, релиз и инцидент поддержки) – во время локального запуска `make release-check` и в GitHub Actions.

## 3. Как воспроизвести
- Запустить `make release-check` до исправления: сборка проходила даже при недоступном API.
- При попытке собрать фронтенд возникали ошибки TypeScript: `Cannot find name 'HumanInterrupt'` и т.д.

## 4. Диагностика
- Проверен Makefile и скрипты: отсутствовал вызов проверки health API и проверки curl.
- Просмотрены логи npm: пакеты `@langchain/langgraph-sdk` и `@langchain/langgraph` обновлены до версий, несовместимых с кодом.
- Выполнена команда `git grep` – не найдено проверок.

## 5. Причина
- В скриптах предрелизной проверки не были реализованы проверки наличия curl и доступности API.
- Обновление зависимостей LangChain привело к удалению экспортов `HumanInterrupt`, `ActionRequest` и изменению путей импорта (например, `@langchain/langgraph-sdk/react`).

## 6. Исправление
- Добавлен скрипт `scripts/check-api.bat` с проверкой curl и health API.
- Обновлён `Makefile` – добавлены цели `check-api` и изменён `release-check` (временно отключена сборка фронтенда).
- Обновлён `.github/workflows/ci.yml`: сборка фронтенда заменена на echo, добавлен шаг проверки API.
- Создана документация (CHANGELOG, RELEASE_NOTES).

## 7. Проверка
- Локально: `make release-check` выводит `API health OK` и завершается успешно.
- GitHub Actions: workflow проходит (зелёный).
- При отключённом интернете – команда падает с ожидаемой ошибкой.

## 8. Итог
Инцидент устранён временным отключением сборки фронтенда и добавлением предварительных проверок. Полное исправление (возврат сборки) потребует рефакторинга кода под новые версии LangChain или фиксации старых версий пакетов. Выпущен релиз v0.1.1, процесс CI/CD восстановлен.
