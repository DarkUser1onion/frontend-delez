# BACKUP_RESTORE_REPORT – Delёз

## Цель
Проверить возможность резервного копирования и восстановления пользовательских данных (конфигурация приложения).

## Что бэкапится
- Директория `~/.config/tech.delez.app` (Linux) / `%APPDATA%\tech.delez.app` (Windows)
- Содержит: настройки интерфейса, тему, язык, параметры уведомлений.

## Как создавался backup
- Скрипт `backup.sh` создаёт копию с временной меткой в папку `backups/`.
- Пример: `backups/config_backup_20260610_025843`

## Как проверялось восстановление
1. Оригинальная директория конфигов удалена: `rm -rf /home/user/tech.delez.app`
2. Восстановление из бэкапа: `cp -r backups/config_backup_20260610_025843 /home/user/tech.delez.app` или скрипт в зависимости от системы.
3. Запуск приложения – настройки сохранены, приложение работает.

## Результат
- Backup успешно создан.
- Restore успешно выполнен.
- Данные не потеряны.

## Рекомендации
- Добавить автоматический backup через cron (раз в день).
- Для Windows использовать планировщик задач с `backup.bat`.
